# 1.3 常用的Map

/**
 * @author Mr.Lee
 * @description:
 * @date 2021-07-02 23:55:29
 */
public class MapExample {
    // Set Use Times: 752; Map Size: 1000000  --- Get Use Time: 2966
//  Map<UUID, UUID> map = new Hashtable<>();

    // Set Use Times: 465; Map Size: 950561  --- Get Use Time: 4552
//  Map<UUID, UUID> map = new HashMap<>();

    // Set Use Times: 735; Map Size: 1000000  --- Get Use Time: 3952
//  Map<UUID, UUID> map = Collections.synchronizedMap(new HashMap<>());

    // Set Use Times: 1185; Map Size: 1000000  --- Get Use Time: 793
    Map<UUID, UUID> map = new ConcurrentHashMap<>();



    public static void main(String[] args) {
        MapExample example = new MapExample();
        example.startPut();

        System.out.println("---------");

        example.startGet();
    }


    public Map<UUID, UUID> getMap() {
        return map;
    }

    final static int COUNT = 1000000;
    final static int THREAD_COUNT = 100;

    // 第个线程装入的容量
    final static int GAP = COUNT / THREAD_COUNT;

    static UUID[] keys = new UUID[COUNT];
    static UUID[] values = new UUID[COUNT];

    static {
        //  准备生成测试数据
        for (int i = 0; i < COUNT; i++) {
            keys[i] = UUID.randomUUID();
            values[i] = UUID.randomUUID();
        }
    }
    
    public UUID getKey(int i) {
        if (i >= COUNT || i < 0) {
            return null;
        }
        return keys[i];
    }

    public UUID getValue(UUID uuid) {
        return getMap().get(uuid);
    }

    /**
     * 测试各Map的写速度
     */
    public void startPut() {
        Thread[] threads = new Thread[THREAD_COUNT];
        for (int i = 0; i < THREAD_COUNT; i++) {
            threads[i] = new PutThread(i * GAP);
        }

        Long start = System.currentTimeMillis();
        for (int i = 0; i < THREAD_COUNT; i++) {
            threads[i].start();
        }
        for (Thread t : threads) {
            try {
                t.join();
            } catch (InterruptedException ignored) {
            }
        }
        Long end = System.currentTimeMillis();
        System.out.println("Use Times: " + (end - start));
        System.out.println("Map Size: " + getMap().size());
    }

    /**
     * 测试各Map的读速度
     */    
    public void startGet() {
        UUID key = keys[10];

        Thread[] threads = new Thread[THREAD_COUNT];
        for (int i = 0; i < THREAD_COUNT; i++) {
            threads[i] = new GetThread(key);
        }
        long start = System.currentTimeMillis();
        for (int i = 0; i < THREAD_COUNT; i++) {
            threads[i].start();
        }
        for (Thread t : threads) {
            try {
                t.join();
            } catch (InterruptedException ignored) {
            }
        }
        long end = System.currentTimeMillis();
        System.out.println("Use Time: " + (end - start));
    }

    /**
     * 写线程
     * 每个线程向map中写入1w对uuid
     */
    class PutThread extends Thread {

        int start;

        public PutThread(int start) {
            this.start = start;
        }

        @Override
        public void run() {
            for (int i = start; i < start + GAP; i++) {
                // 内部类, 可以直接调用外部类的方法
                getMap().put(keys[i], values[i]);
            }
        }
    }

    /**
     * 读线程
     * 每个线程重复从Map中读1w次
     */
    class GetThread extends Thread {
        UUID key;
        public GetThread(UUID key) {
            this.key = key;
        }

        @Override
        public void run() {
            for (int i = 0; i < 10000; i++) {
                getMap().get(key);
            }
        }
    }
}

# 1.3.1 Hashtable

Set Use Times: 752; Map Size: 1000000 --- Get Use Time: 2966

从上面的测试数据中, 可以看到…Hashtable是线程安全的. 在多线程的环境下写入数据, 是可以保证数据的正确性的.

Hashtable中的关键方法, 都是用synchronized修饰的.

# 1.3.2 HashMap

Set Use Times: 465; Map Size: 950561 --- Get Use Time: 4552

从上面的测试数据中, 可以看到…HashMap是线程不安全的. 在多线程的环境下写入数据, 不能保证数据的正确性的.

# 1.3.3 SynchronizedMap

Set Use Times: 735; Map Size: 1000000 --- Get Use Time: 3952

从上面的测试数据中, 可以看到…SynchronizedMap与Hashtable 一样, 是线程安全的. 在多线程的环境下写入数据, 是可以保证数据的正确性的.

与Hashtable不同是的, SynchronizedMap 没有直接锁方法, 而是在方法中锁了mutex对象(看过源码, 后感觉无甚差异. 但是话不能这么说, 毕竟锁的粒度是变小了的)…

# 1.3.4 ConcurrentHashMap

Set Use Times: 1185; Map Size: 1000000 --- Get Use Time: 793

从上面的测试数据中, 可以看到…ConcurrentHashMap, 是线程安全的. 在多线程的环境下写入数据, 是可以保证数据的正确性的.

ConcurrentHashMap 底层采用红黑树存储数据. 所以在写入数据时, 会要比其他的几种Map所用时间要长一些.

查看源码, 可以看到 ConcurrentHashMap, 是采用CAS的方式, 来保证线程的正确性的

static final <K,V> boolean casTabAt(Node<K,V>[] tab, int i, Node<K,V> c, Node<K,V> v) {
    return U.compareAndSetObject(tab, ((long)i << ASHIFT) + ABASE, c, v);
}

# 1.3.5 总结

从上面的测试数据, 可以总结得出结论.

如果是单线程的情况下, 果断使用 HashMap没毛病.

在多线程的环境下:

写入次数 > 读取次数 用SynchronizedMap

读取次数 > 写入次数 用ConcurrentHashMap

还有要考虑的, 线程数量太多时, 也可以考虑使用SynchronizedMap, 毕竟10000个线程在CPU那自旋, CPU也扛不住不是…

根据上面的执行结果, 可以看到有三个问题点:

  • 售出了重复的“票”(如 999号票)
  • 售出了空票
  • 超卖现象(数组越界)

下面根据表现出的这些问题并结合ArrayList源码, 进行分析:

// 当我看到size没有用volatile修饰时, 就意识到问题了(人家ArrayList就不是给多线程环境用的)
private int size;

public int size() {
    return size;
}

public E remove(int index) {
    Objects.checkIndex(index, size);
    final Object[] es = elementData;

    @SuppressWarnings("unchecked") E oldValue = (E) es[index];
    fastRemove(es, index);

    return oldValue;
}

private void fastRemove(Object[] es, int i) {
    modCount++;
    final int newSize;
    if ((newSize = size - 1) > i)
        System.arraycopy(es, i + 1, es, i, newSize - i);
    es[size = newSize] = null;
}

==售出重复==和==售出空票==实际上是同一个问题: 原因都是 ArrayList.remove()不是原子操作

重复的问题:

假设线程A 进入了remove()方法, 在执行过E oldValue = (E) es[index];后, 线程挂起;

线程B 进入了remove()方法, 这时size还没有被修改, 所以检查是可以通过的. 同理, es 也是没有修改过的. 这个时候 线程A 和 线程B 读出的oldValue是相同的.

空票的问题:

假设线程A 进入了remove()方法, 在执行过final Object[] es = elementData;后, 线程挂起;

线程B 进入了remove()方法, 执行到es[size = newSize] = null; 后, 线程挂起;

线程A 继续执行, 从es中读到es[0]为null. 这也就解释了只有当size = newSize == 0时, 才会出现空票现象.

==超卖的问题==

对于这个问题呢…上面的代码里, 已经给出了答案. 原因是: list.size() 和 list.remove(0) 他们俩不是原子操作. 就不能保证数据的正确性.

// 我给出的解决方案是, 让 list.size() 和 list.remove(0) 变成一个原子操作.
synchronized (list) {
    if (list.size() > 0) {
        System.out.println("售出: " + list.remove(0));
    } else {
        break;
    }
}

# 1.4.5 总结

Vector 所有方法都使用了synchronized 但是 size 没有用 volatile 修饰, 用时要注意.

ArrayList 这货, 设计之初的使用场景就是在单独线程中使用吧…但是只要代码写的合理, 也不是不能在多线程环境下使用…完全要看业务场景了.

CopyOnWriteArrayList 写入方法用synchronized(lock)包裹, 适合写少读多的场景.

SynchronizedList 方法都是用synchronized (mutex)包裹, 但是 size 有没有用 volatile 修饰, 这取决于使用的构建对象.

    }

# 2.1.1 ConcurrentSkipListMap(跳表)

在非并发环境下, 可以使用无序的HashMap 和有序的Hashtable

在并发环境下, 可以使用无序的ConcurrentHashMap 和有序的ConcurrentSkipListMap

跳表: 底层是用一个链表来存储数据的, 他是排好顺序的. 链表遍历查找某个元素时, 时间复杂度为O(n). 为解决查找效率问题, 衍生出了跳表 结构. 如上图示: 在底层链表的基础上, 拿出一些关键元素, 在上面再做一层. 以此类推下去.

比如: 我要找元素11 原来我要循环遍历6次, 才能找到. 在跳表 结构中, 我遍历3次就够了(还是二分嘛)

/**
 * @description:
 * @author: Mr.Lee
 * @date: 2021-07-07 06:16

public class CopyOnWriteExample {

    // Write, Use Time: 37, Size: 57766; Read, Use Time: 29
//    List<String> stoup = new ArrayList<>();
    // Write, Use Time: 29, Size: 100000; Read, Use Time: 254
//    List<String> stoup = new Vector<>();
    // Write, Use Time: 2335, Size: 100000; Read, Use Time: 23
    List<String> stoup = new CopyOnWriteArrayList<>();


    final static int GAP = 10000;

    int generateWriteRunnableCount = 0;

    public static void main(String[] args) {
        CopyOnWriteExample example = new CopyOnWriteExample();
        example.write();
        example.read();
    }

    private void write() {
        Thread[] threads = generateThreads(this::getWriteRunnable, 20);
        for (int i = 0; i < 10; i++) {
            int start = i * GAP;
            threads[i] = new Thread(() -> {
                for (int j = start; j < start + GAP; j++) {
                    stoup.add("booze_" + j);
                }
            });
        }

        long startTime = System.currentTimeMillis();
        startThread(threads);
        long endTime = System.currentTimeMillis();
        System.out.print("Write, Use Time: " + (endTime - startTime) + ", Size: " + stoup.size() );
        System.out.print("; ");
    }

    private void read() {
        Thread[] threads = generateThreads(this::getReadRunnable, 10);

        long startTime = System.currentTimeMillis();
        startThread(threads);
        long endTime = System.currentTimeMillis();
        System.out.println("Read, Use Time: " + (endTime - startTime));
    }

    private Thread[] generateThreads(Supplier<Runnable> supplier , int size) {
        Thread[] threads = new Thread[size];
        for (int i = 0; i < size; i++) {
            threads[i] = new Thread(supplier.get());
        }
        return threads;
    }

    private void startThread(Thread[] threads) {
        Arrays.stream(threads).forEach(Thread::start);
        for (Thread thread : threads) {
            try {
                thread.join();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }

    private Runnable getReadRunnable() {
        return () -> {
            for (int j = 0; j < 100 * GAP; j++) {
                stoup.get(55);
            }
        };
    }

    private Runnable getWriteRunnable() {
        int start = generateWriteRunnableCount++;
        return () -> {
            for (int j = start; j < start + GAP; j++) {
                stoup.add("booze_" + j);
            }
        };
    }
}

public class BlockingQueueExample {

    // 无界的
    private static final BlockingQueue<String> infinite = new LinkedBlockingQueue<>();
    private static final Random random = new Random();

    private static volatile int isRunning = 1;

    public static void main(String[] args) {
        Thread producer = new Thread(() -> {
            for (int i = 0; i < 100; i++) {
                try {
                    infinite.put("message_" + i);
                    SleepHelper.sleepMilliSeconds(random.nextInt(500));
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
            isRunning = 0;
        });

        for (int i = 0; i < 5; i++) {
            new Thread(() -> {
                String name = Thread.currentThread().getName();
                System.out.println(name + ", start...");
                while (isRunning == 1) {
                    try {
                        // 因为 take() 当从队列中取不到值时, 会阻塞. 所以Customer永远不会Done
                        System.out.println(name + ", take: " + infinite.take());

                }
                System.out.println(name + ", Done...");
            }, "Customer " + (i + 1)).start();
        }
        SleepHelper.sleepSeconds(2);
        producer.start();
    }
}

/** The capacity bound, or Integer.MAX_VALUE if none */
private final int capacity;

public LinkedBlockingQueue() {
    this(Integer.MAX_VALUE);
}

public LinkedBlockingQueue(int capacity) {
    if (capacity <= 0) throw new IllegalArgumentException();
    this.capacity = capacity;
    last = head = new Node<E>(null);
}

public void put(E e) throws InterruptedException {
    if (e == null) throw new NullPointerException();
    final int c;
    final Node<E> node = new Node<E>(e);
    final ReentrantLock putLock = this.putLock;
    final AtomicInteger count = this.count;
    // put 为原子操作, 给put上锁
    putLock.lockInterruptibly();
    try {
       	// 之所以不用if, 是因为provider解封后, 当前线程拿到执行资源时, 其他线程已经把count填满了, 所以依旧无法执行. 所以不能只判断一次
        while (count.get() == capacity) {
            notFull.await();
        }
        enqueue(node);
        c = count.getAndIncrement();
        if (c + 1 < capacity)
			// 放开take阻塞的线程
            notFull.signal();
    } finally {
        // 结束原子操作, 给put解锁
        putLock.unlock();
    }
    if (c == 0) {
        // 这不可能等于 0 啊~ 没看太明白~~
        signalNotEmpty();
    }
}

/**
 * @author Mr.Lee
 * @description: 写一个固定容量的同步容器, 拥有put和get方法. 以及getCount方法. 能够支持, 2个生产者和10个消费者线程的阻塞调用.
 * @date 2021-07-08 06:52:38
 */
public class Topic_03_02 {

    private static final AtomicInteger x = new AtomicInteger(1);

    public static void main(String[] args) {
        Basket<String> basket = new Basket<>(10);
        List<Thread> providers = new ArrayList<>();
        List<Thread> consumers = new ArrayList<>();
        Runnable providerRunnable = () -> {
            while (true) {
                String o = String.format("O_%05d", x.incrementAndGet());
                basket.put(o);
                SleepHelper.sleepMilliSeconds(1000); // 20/s
                System.out.println(String.format("%s +1....[%s]", Thread.currentThread().getName(), o));
            }
        };
        Runnable consumerRunnable = () -> {
            while (true) {
                Object o = basket.get();
                SleepHelper.sleepMilliSeconds(500); // 20/s
                System.out.println(String.format("%s -1....[%s]", Thread.currentThread().getName(), o));
            }
        };
        for (int i = 0; i < 2; i++) {
            providers.add(new Thread(providerRunnable, String.format("provider_%02d", i + 1)));
        }

        for (int i = 0; i < 10; i++) {
            consumers.add(new Thread(consumerRunnable, String.format("consumer_%02d", i + 1)));
        }


        providers.forEach(Thread::start);
        SleepHelper.sleepSeconds(1);
        consumers.forEach(Thread::start);
    }
	
    private static class Basket<T> {

        private BlockingQueue<T> queue;

        public Basket(int capacity) {
            // 也是一样可以实现有界的阻塞队列. 一个用链表存储, 一个用数组存储
//            queue = new LinkedBlockingQueue<>(capacity);
            queue = new ArrayBlockingQueue<>(capacity);

        }

        public void put(T o) {
            try {
                queue.put(o);
            } catch (InterruptedException e) {
                System.out.println("写入阻塞的线程, 被打断");
            }
        }

        public T get() {
            try {
                return queue.take();
            } catch (InterruptedException e) {
                System.out.println("读取阻塞的线程, 被打断");
            }
            return null;
        }
    }
}

# 2.3.1.4 DelayQueue

自己定义的优先级队列

他不像常规的队列, 队列的头部是固定,不能变化的, DelayQueue是一个可以根据需求调整, 在队列中的调度顺序

/**
 * @author Mr.Lee
 * @description:
 * @date 2021-07-08 22:08:04
 */
public class DelayQueueExample {

    private static final BlockingQueue<Task> tasks = new DelayQueue<>();

    public static void main(String[] args) throws InterruptedException {
        long now = System.currentTimeMillis();
        tasks.put(new Task("Task 1", now + 100));
        tasks.put(new Task("Task 2", now + 200));
        tasks.put(new Task("Task 3", now + 300));
        tasks.put(new Task("Task 4", now + 400));
        tasks.put(new Task("Task 5", now + 111));

        System.out.println(tasks);

        for (int i = 0; i < 5; i++) {
            System.out.println(tasks.take());
        }
    }

    private static class Task implements Delayed {

        private String name;
        private Long runningTime;

        public Task(String name) {
            this.name = name;
            this.runningTime = System.currentTimeMillis();
        }

        public Task(String name, Long runningTime) {
            this.name = name;
            this.runningTime = runningTime;
        }

        @Override
        public long getDelay(TimeUnit unit) {
            // 获得任务调度优先权的比较值
            return unit.convert(runningTime - System.currentTimeMillis(), MILLISECONDS);
        }

        @Override
        public int compareTo(Delayed o) {
            // 任务调度优先级, 根据 getDelay() 从小到大排列
            if (this.getDelay(MILLISECONDS) < o.getDelay(MILLISECONDS)) {
                return -1;
            } else if (this.getDelay(MILLISECONDS) > o.getDelay(MILLISECONDS)) {
                return 1;
            } else {
                return 0;
            }
        }

        @Override
        public String toString() {
            return "Task{" +
                    "name='" + name + '\'' +
                    ", runningTime=" + runningTime +
                    '}';
        }
    }
}

/**
 * @author Mr.Lee
 * @description: 自定义优先级的队列
 * @date 2021-07-08 23:11:18
 */
public class DelayQueueExample_02 {

    static boolean isPrint = false;
    private static final BlockingQueue<Task> tasks = new DelayQueue<>();

    public static void main(String[] args) throws InterruptedException {
        long now = System.currentTimeMillis();
        tasks.put(new Task("Task 1", 1L));
        tasks.put(new Task("Task 2", 2L));
        tasks.put(new Task("Task 3", 3L));
        tasks.put(new Task("Task 4", 4L));
        tasks.put(new Task("Task 5", 5L));

//      System.out.println(tasks);
//        isPrint = true;
        for (int i = 0; i < 5; i++) {
            System.out.println(tasks.take());
            if (i == 0) {
                // 模拟调整优先级
                TaskStatus.name = "Task 5";
                isPrint = true;
            } else {
                isPrint = false;
            }
        }
    }

    private static class Task implements Delayed {

        private final String name;
        private final Long runningTime;

        public Task(String name) {
            this.name = name;
            this.runningTime = System.currentTimeMillis();
        }

        public Task(String name, Long runningTime) {
            this.name = name;
            this.runningTime = runningTime;
        }

        @Override
        public long getDelay(TimeUnit unit) {
            // 获得任务调度优先权的比较值

            long diff = runningTime - System.currentTimeMillis();
            // 可以看到, 队列每一次修改, 都会重新调用getDelay方法. 也就是说, 会给队列重新排序(这一点很重要).
            // 可能执行完一个任务后, 原来排在后面的任务, 权重增加, 就可以用Delay来实现了
            // 模拟调整优先级
            if (this.name.equals(TaskStatus.name)) {
                // 默认队首项, 会 默认 * 1000000 要在默认项前面执行, 就要比他还小才行
                diff = -(System.currentTimeMillis() * 1000000 + 1);
            }
            long x = unit.convert(diff, MILLISECONDS);

            if (isPrint) {
                System.out.println(name + ", Delay: " + x);
            }
            return x;
        }

        @Override
        public int compareTo(Delayed o) {
            // 任务调度优先级, 根据 getDelay() 从小到大排列
            if (this.getDelay(MILLISECONDS) < o.getDelay(MILLISECONDS)) {
                return -1;
            } else if (this.getDelay(MILLISECONDS) > o.getDelay(MILLISECONDS)) {
                return 1;
            } else {
                return 0;
            }
        }

        @Override
        public String toString() {
            return "Task{" +
                    "name='" + name + '\'' +
                    ", runningTime=" + runningTime +
                    '}';
        }
    }

    /**
     * 模拟任务状态
     */
    private static class TaskStatus {
        private static String name = "";
    }
}
