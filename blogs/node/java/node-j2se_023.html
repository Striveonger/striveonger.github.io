Java 并发编程(一) - 多线程基础

Striveonger 2021-08-06 21:26:00 JavaConcurrent

多线程基础

# 1. 并发与并行

# 1. 并发

定义: 在操作系统中, 在同一个时间段内, 两个或多个程序执行，有时间上的重叠(宏观上是同时, 微观上仍是顺序执行)

只是, 时间间隔太小, 从宏观上感知, 几个任务在同时执行, 微观上的几个任务是在被CPU轮流执行.

# 1.1.1 高并发的案例

系统集中收到了大量的请求, 比如12306的抢票, 双11秒杀.

它会导致系统在短时间内执行大量的操作. 例如对资源的请求，数据库的操作等

如果高运发处理不好不仅仅降低了用户的体验度, 请求时间变长, 也可能导致系统宕机, 严重的甚至导致OOM异常, 系统停止工作等

若想系统能够适应高并发的状态, 就需要从多个方面进行系统优化:

  • 提升硬件性能
  • 网络系统架构开发语言的选取
  • 数据结构的运用
  • 算法优化
  • 数据库优化
  • ...

# 1.2 并行

定义: 在操作系统中, 一组程序按独立异步的速度执行, 无论从微观还是宏观, 程序都是一起执行的.

一般多处在, 多CPU或CPU多核心的机器上会出现并行.

# 1.3 对比

顾名思义，并发(Concurrent) 强调的是可以一起「出『发』」, 「并行」(Sequential)强调的是可以一起「执『行』」.

串行: 有一个任务执行单元, 从物理上就只能一个任务, 一个任务地执行 并行: 有多个任务执行单元, 从物理上就可以多个任务一起执行

也就是说, 在任意时间点上, 串行执行时必然只有一个任务在执行, 而并行则不一定.

# 2. 线程概念

定义: 线程是一个程序中不同的执行路径

# 2.1. 线程与进程

进程(process) 和 线程(thread) 是操作系统的基本概念.

**进程: ** 是一个容器(静态)

是资源分配的基本单位

**线程: ** 是容器中的工作单位(实际的工作单元)

调度执行的基本单位, 多个线程共享资源

进程中包含一个或多个线程.

当启动一个进程时, 实际上是在启动主线程启动.

# 2.2 创建线程

  • 继承 Thread类
  • 实现 Runnable接口
/**
 * @author Mr.Lee
 * @description: 
 * @date 2021-04-10 1:14 PM
 */
public class CreateThreadExample {

    public static void main(String[] args) throws Exception {
        // 1. 继承 Thread类
        T t1 = new T("t1");
        t1.start();

        // 2. 实现 Runnable接口
        Runnable runnable = () -> System.out.println(Thread.currentThread().getName() + " ruing...");
        Thread t2 = new Thread(runnable, "t2");
        t2.start();

        // 3. 使用线程池的方式(也是利用Worker调用其内部Runnable的run()方法)
        ExecutorService service = Executors.newCachedThreadPool();
        service.execute(runnable);

        // 4. 实现 Callable接口
        // 是把Callable对象封装成了RunnableFuture对象, 依旧还是当Runnable来执行的
        Callable<String> callable = () -> Thread.currentThread().getName() + "; Callable Finish...";
        Future<String> future = service.submit(callable);
        System.out.println(future.get());
        service.shutdown();

        // 5. FutureTask(做为RunnableFuture的实现来说, 哈哈~同上)
        FutureTask<String> task = new FutureTask<>(callable);
        new Thread(task).start();
        System.out.println(task.get());
    }

}

class T extends Thread {

  	public T(String name) {
        super(name);
    }

    @Override
    public void run() {
        System.out.println(super.getName() + " ruing...");
    }
}

# 2.3 暂停线程

# 2.3.1 sleep

Java中线程的暂停是调用java.lang.Thread类的sleep方法(注意是类方法). 该方法会使当前正在执行的线程暂停指定的时间, 如果线程持有锁, sleep 方法并不会释放该锁.

public class Main {
    public static void main(String[] args) {
        for (int i = 0; i < 10; i++) {
            System.out.print(i + " ");
            try {
                Thread.sleep(1000);    //当前main线程暂停1000ms
            } catch (InterruptedException e) {
            }
        }
    }
}

# 2.3.2 suspend(废弃)

暂停时，持有的锁是不会释放的。 易造成数据不一致和死锁的问题。

public class ThreadSuspendExample {
    public static void main(String[] args) {
        // 规则: arr[1] - a[0] == 1
        final int arr[] = {0, 1};
        Thread t2 = new Thread(() -> {
            while (true) {
                arr[0] = arr[1];
                SleepHelper.sleepSeconds(1);
                arr[1]++;
                String name = Thread.currentThread().getName();
                System.out.println(StrUtil.format("{}, go on", name));
            }
        }, "t2");

        t2.start();
        SleepHelper.sleepSeconds(2);
        // 暂停线程
        t2.suspend();
        // 如果在这时候, 读取数据. 就会出现不符合规则的情况出现.
        System.out.println(Arrays.toString(arr));
        SleepHelper.sleepSeconds(2);
        // 继续线程
        t2.resume();
    }
}
// 执行结果:
//   t2, go on
//   [2, 2]
//   t2, go on
//   ...

        // 规则: arr[1] - a[0] == 1
        final int arr[] = {0, 1};
        Thread t1 = new Thread(() -> {
            while (true) {
                String name = Thread.currentThread().getName();

        t1.stop();
        System.out.println(Arrays.toString(arr));
    }
}

public class ThreadEndExample {

    private static volatile boolean running = true;
    // 规则: a - b == 1
    private static volatile int a = 0, b = 1;

    public static void main(String[] args) {
        // 优雅的停止线程 1
        a = 0; b = 1;
        Thread t3 = new Thread(() -> {

                b = b + 1;

            System.out.println(Thread.currentThread().getName() + " end...");
        }, "t3");
        t3.start();
        SleepHelper.sleepSeconds(3);
        running = false;
        SleepHelper.sleepSeconds(1);
        System.out.println(a + ", " + b);
    }
}
// 执行结果
//   t3 running...
//   t3 running...
//   t3 end...
//   2, 3

# 2.4.3 interrupt

/**
 * @author Mr.Lee
 * @description: 线程中止
 * @date 2021-04-10 8:01 PM
 */
public class ThreadEndExample {

    private static volatile boolean running = true;
    // 规则: a - b == 1
    private static volatile int a = 0, b = 1;

    public static void main(String[] args) {
        // 优雅的停止线程 2
        a = 0; b = 1;
        Thread t4 = new Thread(() -> {
            while (!Thread.interrupted()) {
                System.out.println(Thread.currentThread().getName() + " running...");
                a = b;
                try {
                    Thread.sleep(2000);
                } catch (InterruptedException e) {
                    // 线程被打断, 模拟业务回退
                    a = a - 1;
                    // 释放资源, 并退出线程
                    break;
                }
                b = b + 1;
            }
            System.out.println(Thread.currentThread().getName() + " end...");
        }, "t4");
        t4.start();
        SleepHelper.sleepSeconds(3);
        t4.interrupt();
        SleepHelper.sleepSeconds(1);
        System.out.println(a + ", " + b);
    }
}
// 执行结果
//   t4 running...
//   t4 running...
//   t4 end...
//   1, 2

    private static volatile int a = 0, b = 1;

    public static void main(String[] args) {
        // 优雅的停止线程 4
        a = 0; b = 1;

}

# 3. 线程状态

状态 描述
NEW 初始状态, 线程对象被创建, 还没有调用start() 方法.
RUNNABLE 运行状态, Java线程将操作系统中的就绪和运行两种状态笼统地称作"运行中"
BLOCKED 阻塞状态, 表示线程阻塞于锁(synchronized)
WAITING 等待状态, 表示需要其他线程显示唤醒该线程
TIMED_WAITING 计时等待状态, 表示在指定的时间后, 自己唤醒该线程
TERMINATED 终止状态, 表示当前线程已执行完毕.
/**
 * @author Mr.Lee
 * @description: 线程状态的例子
 * @date 2021-04-10 2:01 PM
 */
public class ThreadStateExample {

    public static void main(String[] args) throws Exception {

        // ====================================================================
        Thread t1 = new Thread(() -> {
            Thread current = Thread.currentThread();
            System.out.println(StrUtil.format("2. name: {}, state: {}", current.getName(), current.getState()));
            for (int i = 0; i < 3; i++) {
                SleepHelper.sleepSeconds(1);
                System.out.print(i + " ");
            }
            System.out.println();
        }, "t1");
        System.out.println(StrUtil.format("1. name: {}, state: {}", t1.getName(), t1.getState()));
        t1.start();
        t1.join();
        System.out.println(StrUtil.format("3. name: {}, state: {}", t1.getName(), t1.getState()));
        System.out.println();

        // ====================================================================
        Thread t2 = new Thread(() -> {
            // 挂起
            LockSupport.park();
            System.out.println("t2 go on...");
            SleepHelper.sleepSeconds(5);
        }, "t2");
        t2.start();
        SleepHelper.sleepSeconds(1);
        System.out.println(StrUtil.format("4. name: {}, state: {}", t2.getName(), t2.getState()));
        // 唤醒
        LockSupport.unpark(t2);

        SleepHelper.sleepSeconds(1);
        System.out.println(StrUtil.format("5. name: {}, state: {}", t2.getName(), t2.getState()));
        System.out.println();

        // ====================================================================
        final Object o = new Object();
        Thread t3 = new Thread(() -> {
            synchronized (o) {
                System.out.println("t3 have lock");
            }
        }, "t3");

        new Thread(() -> {
            synchronized (o) {
                SleepHelper.sleepSeconds(5);
            }
        }).start();

        SleepHelper.sleepSeconds(1);
        t3.start();
        SleepHelper.sleepSeconds(1);
        System.out.println(StrUtil.format("6. name: {}, state: {}", t3.getName(), t3.getState()));
        t3.join();
        System.out.println();

        // ====================================================================
        final Lock lock = new ReentrantLock(); // JUC 锁, 使用CAS的实现
        Thread t4 = new Thread(() -> {
            lock.lock();
            System.out.println("t4 have lock");
            lock.unlock();
        }, "t4");

        new Thread(() -> {
            // 不会进入 BLOCKED, 而是 WAITING 状态
            lock.lock();
            SleepHelper.sleepSeconds(5);
            lock.unlock();
        }).start();

        SleepHelper.sleepSeconds(1);
        t4.start();
        SleepHelper.sleepSeconds(1);
        System.out.println(StrUtil.format("7. name: {}, state: {}", t4.getName(), t4.getState()));
        t4.join();
        System.out.println();

        // ====================================================================
        Thread t5 = new Thread(LockSupport::park, "t5");
        t5.start();
        SleepHelper.sleepSeconds(1);
        System.out.println(StrUtil.format("8. name: {}, state: {}", t5.getName(), t5.getState()));
        LockSupport.unpark(t5);
        System.out.println();
    }
}

# 3.2 可运行状态(RUNNABLE)

线程执行start()的状态, 在就绪状态和运行状态反复横跳(取决于系统的调度)

# 3.3 阻塞状态(BLOCKED)

当一个线程试图获取一个内部的对象锁(而不是 java.util.concurrent 库中的锁), 而该锁被其他线程持有时, 则该线程进人阻塞状态

/**
 * @author Mr.Lee
 * @description:
 * @date 2021-04-03 10:24 AM
 */
public class ThreadExample {

    protected static volatile AtomicInteger x = new AtomicInteger(0);

    public static void main(String[] args) throws Exception {

        Thread t1 = new Thread(() -> {
          	// 持有锁
            print();
            x.incrementAndGet();
            System.out.println("t1: " + x.get());
        });

        Thread t2 = new Thread(() -> {
            // 让出执行权, 保证让t1拿到锁
            Thread.yield();
            // 进入阻塞状态
            print();
            x.incrementAndGet();
            System.out.println("t2: " + x.get());
        });
        t1.start();
        t2.start();

        Thread.sleep(10);
        System.out.println("t1 state: " + t1.getState());
        System.out.println("t2 state: " + t2.getState());

        while (x.get() < 2) {
            // 主线程让出执行权
            Thread.yield();
        }
        System.out.println();
    }

    private static synchronized void print() {
        try {
            // 这是模拟一个耗时操作
            List<Object> list = new ArrayList<>();
            for (int i = 0; i < 5000000; i++) {
                list.add(new Object());
            }
        } catch (Exception ignored) { }
        System.out.println("exec print...");
    }
}

# 3.4 等待状态 / 计时等待状态

阻塞和等待的区别在于, 阻塞是被动的, 它是在等待获取 monitor lock. 而等待是主动的, 通过调用 Object.wait() 等方法进入.

进入方法 退出方法
Object.wait() Object.notify() / Object.notifyAll()

 * @description:
 * @date 2021-04-03 10:24 AM

public class ThreadExample {

    protected static volatile AtomicInteger x = new AtomicInteger(0);

    public static void main(String[] args) throws Exception {
        // 线程间, 共用的锁对象
        final Object lock = new Object();

        Thread t1 = new Thread(() -> {
            synchronized (lock) {
                try {
                    for (int i = 0; i < 5; i++) {
                        System.out.println(Thread.currentThread().getName() + "; i = " + i);
                    }
                    // 会使 t1 进入 TIMED_WAITING 状态
                    TimeUnit.SECONDS.sleep(2);
                    x.incrementAndGet();
                    System.out.println("t1: " + x.get());
                    // 唤醒被挂起的 t2 线程
                    lock.notify();
                } catch (Exception ignored) { }
            }
        }, "t1");

        Thread t2 = new Thread(() -> {
            synchronized (lock) {
                for (int j = 0; j < 5; j++) {
                    try {
                        if (j == 3) {
                            // 挂起 t2 线程, 会释放所持有的锁, 在被唤醒后, 后重新竞争所要持有的锁.
                            lock.wait(50);
                        }
                    } catch (Exception e) {
                    }
                    System.out.println(Thread.currentThread().getName() + "; j = " + j);
                }
                x.incrementAndGet();
                System.out.println("t2: " + x.get());
            }
        }, "t2");

        t2.start();
        Thread.sleep(10);
        t1.start();

        Thread.sleep(100);
        System.out.println("t1 state: " + t1.getState());
        System.out.println("t2 state: " + t2.getState());

        while (x.get() < 2) {
            // 主线程让出执行权
            Thread.yield();
        }
        System.out.println();
    }
}

# 4. 线程调度

# 4.1 sleep方法

sleep方法, 会导致当前线程休眠. 但是sleep不会施放当前线程持有的锁

# 4.2 wait 方法

wait方法, 会主动让出当前线程持有的锁,并挂起当前线程. 当该线程被其他线程唤醒时, 会重新竞争所要持有的锁

# 4.3 yield 方法

yield方法, 会让出当前线程的执行权, 交由CPU重新选择要执行的线程

# 4.4 join 方法

join方法, 会使当前线程转为阻塞状态, 等到另一个线程结束. 当前线程再由阻塞状态变为可运行状态

# 4.5 notify/notifyAll 方法

notify/notifyAll 方法, 会唤醒一个/多个处于挂起状态的线程

/**
 * @author Mr.Lee
 * @description:
 * @date 2021-04-03 10:24 AM
 */
public class ThreadExample {

    protected static volatile AtomicInteger x = new AtomicInteger(0);

    public static void main(String[] args) throws Exception {
        // 线程间, 共用的锁对象
        final Object lock = new Object();

        Thread t1 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println(Thread.currentThread().getName() + "; i = " + i);
                try {
                    Thread.sleep(1000);
                } catch (Exception ignored) { }
            }
            x.incrementAndGet();
        }, "t1");

        Thread t2 = new Thread(() -> {
            for (int i = 0; i < 5; i++) {
                System.out.println(Thread.currentThread().getName() + "; i = " + i);
                if (i == 2) {
                    try {
                      	// 使当前线程挂起, 直到 t1 执行结束, t2 才会被唤醒
                        t1.join();
                    } catch (Exception ignored) { }
                }
            }
            try {
                Thread.sleep(1000);
            } catch (Exception ignored) { }
            x.incrementAndGet();
        }, "t2");

        t1.start();
        t2.start();
        Thread.sleep(10);

        System.out.println("t1 state: " + t1.getState());
        System.out.println("t2 state: " + t2.getState());

        // CAS
        while (x.get() < 2) {
            // 主线程让出执行权
            Thread.yield();
        }
        System.out.println();
        System.out.println("t1 state: " + t1.getState());
        System.out.println("t2 state: " + t2.getState());
    }
}
