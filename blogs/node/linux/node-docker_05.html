# 容器数据卷

数据? 如果数据也在容器中, 那么我们容器删除时, 数据也就会丢失. 需求: ==数据可以持久化==

比如: MySQL容器, 可以将数据可以存储在本地.

容器间, 也可以有一个数据共享的技术, Docker容器中产生的数据, 同步到本地.

这就是卷技术.

数据卷的出现, 是为了解决==容器的持久化与同步操作, 和容器间的数据共享==

# Dockerfile

Dockerfile 是用来构建Docker镜像的命令脚本文件.

# Dockerfile 介绍

构建步骤:

  1. 编写一下Dockerfile文件
  2. docker build 构建成一个镜像
  3. docker run 运行自己的镜像
  4. docker push 发布自己的镜像(Docker Hub, 阿里云)

There are a thousand Hamlets in a thousand people's eyes. 通常我们会根据自己的需求, 来构建我们自己的镜像.

# Dockerfile 构建过程

# 基础知识

  1. 每个保留关键字(指令), 都必须是大写的
  2. 从上到下顺序执行
  3. # 表示注视
  4. 每一个指令, 都会创建一个新的镜像层, 并提交

Dockerfile 是面向开发的. 我们以后要发布项目, 就需要编写Dockerfile文件.

Docker镜像, 逐渐成为企业的交付标准

现在, 大公司...Docker 是做为交付标准存在的.

原来交付 jar包...现在就有交付Docker镜

# Dockerfile 相关指令

  • FROM ==> 基础镜像, 一般会用centos
  • MAINTAINER ==> 镜像的作者, 一般是 姓名 + 邮箱
  • RUN ==> 镜像构建时需要运行的命令, 一般是运行一些安装所需软件的命令
  • ADD ==> 添加一些外部文件, 压缩包会自动解压哦~
  • WORKDIR ==> 设置镜像的工作目录
  • VOLUME ==> 设置容器的挂载卷
  • EXPOSE ==> 设置容器对外暴露的端口
  • CMD ==> 指定容器启动时要执行的命令, 只有最后一个会生效, 可以被替代
  • ENTRYPOINT ==> 指定容器启动时要执行的命令, 可以追加命令
  • ONBUILD ==> 当构建一个被继承的Dockerfile时, 就是运行ONBUILD中的命令(触发指令)
  • COPY ==> 类似 ADD 将文件复制到镜像中
  • ENV ==> 构建时, 设置环境变量

# 子网掩码

定义: 它由32位二进制数组成, 通过运算来指定IP地址中标识网络地址的部分.

作用: 通过与IP地址进行与运算 将某IP地址划分成==网络地址==和==主机地址==

# 算法举例

用上面ens33 网卡举例: 10.13.144.104/24

/24代表, 子网掩码由前24位都是1, 后面都是0的数组成.

换算成二进制: 11111111.11111111.11111111.00000000

换算成十进制: 255.255.255.0

把IP: 10.13.144.104 换算成二进制: 00001010.00001101.10010000.01101000

  1. 获取网络地址

子网掩码 & IP => 00001010.00001101.10010000.00000000 => 10.13.144.0

  1. 获取主机地址

~子网掩码 & IP => 00000000.00000000.00000000.01101000 => 0.0.0.104

子网掩码先取反, 再跟IP进行与运算

# Docker Compose

# 简单了解定义

Docker Compose是用来管理容器工具. 可以轻松的定义和运行多个容器.

官方定义:

Compose is a tool for defining and running multi-container Docker applications. With Compose, you use a YAML file to configure your application’s services. Then, with a single command, you create and start all the services from your configuration. To learn more about all the features of Compose, see the list of features.

Compose works in all environments: production, staging, development, testing, as well as CI workflows. You can learn more about each case in Common Use Cases.

Using Compose is basically a three-step process:

  1. Define your app’s environment with a Dockerfile so it can be reproduced anywhere.
  2. Define the services that make up your app in docker-compose.yml so they can be run together in an isolated environment.
  3. Run docker compose up and the Docker compose command starts and runs your entire app. You can alternatively run docker-compose up using the docker-compose binary.

# Compose 中重要的概念

  • 服务 services: 容器, 应用(MySQL, Redis, Tomcat, 自己的写的SpringBoot应用)
  • 项目 project: 一组关联的容器. 博客(web, mysql)
