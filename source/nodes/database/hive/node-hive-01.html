Hive 查询时莫名奇妙的报错

Mr.Lee 2020-04-30 00:00:23 HiveDocker

Hive遇到FAILED: Execution Error, return code 2 from org.apache.hadoop.hive.ql.exec.mr.MapRedTask错误

# 起因:

Hive数据源, 要测试Kerberos认证方式. 在开启Kerberos认证之前, 是可以正常查询的.

在开启Kerberos认证之后, 通过Kerberos认证方式, 查询也是没有问题的.

在关闭Kerberos认证后, 尝试用普通认证方式查询, 仍然可以正常查询.

但在关闭Kerberos认证情况下, 尝试用Kerberos认证方式查询, 报认证失败后, 再次用普通认证方式查询时, 就会报FAILED: Execution Error, return code 2 from org.apache.hadoop.hive.ql.exec.mr.MapRedTask 错误.

这个报错还具有偶发性.

# 解决:

据排查, 报错原因是 项目中所查询的库由Spark所创建. 但是我使用hive 用户查询Hive时, hive用户, 没有访问所要查询表目录的权限.

修改用户权限后, 仍旧报错.

查看Spark Job的执行日志, 看到以下的报错信息:

Application application_1581349098902_0008 failed 2 times due to AM Container for appattempt_1581349098902_0008_000002 exited with exitCode: -1000
For more detailed output, check application tracking page:http://namenode01:8088/proxy/application_1581349098902_0008/Then, click on links to logs of each attempt.
Diagnostics: Not able to initialize app directories in any of the configured local directories for app application_1581349098902_0008
Failing this attempt. Failing the application.
1
2
3
4

这个其实是一个权限问题，在yarn初始化作业的时候就报错了。是CDH去除kerberos的时候，未重新更新目录权限，如果之前在kerberos环境下已经用过yarn，已经存在对应的目录。去除kerberos之后，未更新权限，导致新作业失败。未启用kerberos前，/yarn/nm/usercache/test目录权限为yarn:yarn，启用后变成test:yarn，权限不兼容，导致yarn生成日志失败，故作业失败。

找到问题就好办啦...

# 解决办法

  1. 查找 NodeManager节点

  2. 获取 yarn.nodemanager.local-dirs

  3. 到NodeManager的yarn.nodemanager.local-dirs，删除usercache目录

    rm -rf /data01/yarn/nm/usercache/*
    1

  4. 重启 yarn组件
