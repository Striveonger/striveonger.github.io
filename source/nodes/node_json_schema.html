对JSON Schema 使用场景的探究

Mr.Lee 2021-03-08 11:06:20 JSONJSON Schema

对JSON Schema 使用场景的探究, 基于 Draft-07

JSON 作为通用的前后端交互，或者后台服务间通信的通用格式被大家广泛使用。我们肯定遇到过一些场景需要校验调用方传递过来的数据格式，比如一定要包含某些字段，某个字段一定要符合某种格式，比如定义了价格的字段，范围一定要在100~200之间，协议字段一定要是TCP或者UDP等枚举类型。你是否在你的用户代码里面自行实现这些判断逻辑呢？如果这样的规则越来越多是不是会显得代码很臃肿呢？这就是为什么要介绍我们今天的主角JSON Schema。JSON Schema定义了JSON格式的规范，各种语言都有开源的第三方JSON Schema校验库，例如Java就有everit-org/json-schema，这样我们就可以定义一份JSON Schema，然后系统的各个模块都可以复用这套JSON规范，不满足规则的数据JSON Schema会直接报错。

在上面的例子中, base类型的widget是没有widgetId的, 但是dynamic类型的widget是有widgetId的. 针对这种情况, 我们如何通过JsonSchema进行校验呢

{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "definitions": {
    "widget_base": {
      "type": "object",
      "properties": {
        "widgetName": {
          "type": "string"
        },
        "chartType": {
          "type": "string"
        },
        "cptConfig": {
          "type": "object",
          "properties": {
            "chartWidth": {
              "type": "integer"
            },
            "Size": {
              "type": "string"
            },
            "backgroundSize": {
              "type": "string"
            },
            "singleUrl": {
              "type": "string"
            },
            "chartHeight": {
              "type": "integer"
            }
          },
          "additionalProperties": false,
          "required": [
            "chartWidth",
            "Size",
            "backgroundSize",
            "singleUrl",
            "chartHeight"
          ],
          "minProperties": 5,
          "maxProperties": 5
        },
        "x": {
          "type": "integer"
        },
        "y": {
          "type": "integer"
        },
        "z": {
          "type": "integer"
        },
        "createTime": {
          "type": "string",
          "pattern": "^\\d{4}(-)(1[0-2]|0?\\d)\\1([0-2]\\d|\\d|30|31) (?:[01]\\d|2[0-3]):[0-5]\\d:[0-5]\\d$"
        },
        "updateTime": {
          "type": "string",
          "pattern": "^\\d{4}(-)(1[0-2]|0?\\d)\\1([0-2]\\d|\\d|30|31) (?:[01]\\d|2[0-3]):[0-5]\\d:[0-5]\\d$"
        },
        "deleted": {
          "type": "integer"
        }
      }
    },
    "widget_needful": {
      "type": "object",
      "oneOf": [
        {
          "properties": {
            "type": {
              "type": "string",
              "enum": ["base"]
            }
          },
          "required": ["type"]
        },
        {
          "properties": {
            "widgetId": {
              "type": "integer",
              "minimum": 1
            },
            "type": {
              "type": "string",
              "enum": ["dynamic"]
            }
          },
          "required": ["widgetId", "type"]
        }
      ]
    }
  },
  "type": "object",
  "properties": {
    "boardsConfig": {
      "type": "object",
      "properties": {
        "boardsBgColor": {
          "type": "string",
          "minLength": 7,
          "maxLength": 7
        },
        "boardsHeight": {
          "type": "integer",
          "minimum": 108,
          "maximum": 108
        },
        "colorAdaption": {
          "type": "boolean"
        },
        "boardsWidth": {
          "type": "integer",
          "minimum": 192,
          "maximum": 192
        },
        "boardsBgImage": {
          "type": "string",
          "minLength": 40,
          "maxLength": 40
        }
      },
      "additionalProperties": false,
      "required": [
        "boardsBgColor",
        "boardsHeight",
        "colorAdaption",
        "boardsWidth",
        "boardsBgImage"
      ],
      "minProperties": 5,
      "maxProperties": 5
    },
    "widgets": {
      "type": "array",
      "items": {
        "allOf": [
          {"$ref": "#/definitions/widget_needful"},
          {"$ref": "#/definitions/widget_base"}
        ]

      }
    }
  },
  "additionalProperties": false,
  "required": [
    "boardsConfig",
    "widgets"
  ],
  "minProperties": 2,
  "maxProperties": 2
}

# $schema

它还声明了该模式所针对的JSONSchema标准的哪个版本. 例如:

  • http://json-schema.org/draft/2019-09/schema#
  • http://json-schema.org/draft-07/schema#
  • http://json-schema.org/draft-06/schema#
  • http://json-schema.org/draft-04/schema#

# type

类型:

  • string

  • Numeric types

    • integer
    • number
    • Multiples
    • Range

  • object

  • array

  • boolean

  • null

# additionalProperties

表示 是否可以添加额外的属性

# properties

表示: object 类型下的属性

# required

表示: object 类型下必须出现的属性

# enum

表示: 该属性为常量, 其value必须在定义中

# items

表示: array 类型下的item

# definitions

变量定义, 用于定义通用的结构. 通常与$ref 配合使用.

在上面的例子中, 定义了两个对象 widget_base 和 widget_needful

# $ref

引用 definitions 中定义的对象.

# Combining schemas

  • allOf: 必须对所有条件有效
  • anyOf: 须对任何一个或多个条件有效
  • oneOf: 必须只有一个条件有效

上述这些关键字都必须设置为array, 其中每个项都是条件

  • not: 对给定条件无效

not 可以做用于 object 中

{
    "widget_needful":{
        "type":"object",
        "oneOf":[
            {
                "properties":{
                    "type":{
                        "type":"string",
                        "enum":[
                            "base"
                        ]
                    }
                },
                "required":[
                    "type"
                ]
            },
            {
                "properties":{
                    "widgetId":{
                        "type":"integer",
                        "minimum":1
                    },
                    "type":{
                        "type":"string",
                        "enum":[
                            "dynamic"
                        ]
                    }
                },
                "required":[
                    "widgetId",
                    "type"
                ]
            }
        ]
    }
}

这里, 生成JsonSchema用的是json-schema-inferrer 校验JsonSchema用的是everit-org/json-schema

package xyz.striveonger.demo.json.schema;

import com.fasterxml.jackson.databind.JsonNode;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.saasquatch.jsonschemainferrer.*;
import org.everit.json.schema.Schema;
import org.everit.json.schema.ValidationException;
import org.everit.json.schema.loader.SchemaLoader;
import org.json.JSONObject;
import org.json.JSONTokener;
import org.junit.Test;

import java.util.EnumSet;

/**
 * @author Mr.Lee
 * @description:
 * @date 2021-03-03 下午2:42
 */
public class JSONSchemaTest {

    @Test
    public void testGenerateSchema() throws Exception {
        ObjectMapper mapper = new ObjectMapper();
        JsonSchemaInferrer inferrer = JsonSchemaInferrer.newBuilder()
                .setSpecVersion(SpecVersion.DRAFT_07)
                // Requires commons-validator
                .addFormatInferrers(FormatInferrers.dateTime()) //, FormatInferrers.email()
                .setAdditionalPropertiesPolicy(AdditionalPropertiesPolicies.notAllowed())
                .setRequiredPolicy(RequiredPolicies.commonFields())
                .setStringLengthFeatures(EnumSet.allOf(StringLengthFeature.class))
                .setNumberRangeFeatures(EnumSet.allOf(NumberRangeFeature.class))
                .setArrayLengthFeatures(EnumSet.allOf(ArrayLengthFeature.class))
                .setObjectSizeFeatures(EnumSet.allOf(ObjectSizeFeature.class))
                .addEnumExtractors(EnumExtractors.validEnum(java.time.Month.class),
                        EnumExtractors.validEnum(java.time.DayOfWeek.class))
                .build();

        // 生成JsonSchema
        JsonNode node = mapper.readTree(JSONSchemaTest.class.getResourceAsStream("/product_invalid.json"));
//        System.out.println(node);
        JsonNode schema = inferrer.inferForSample(node);
        System.out.println(schema);
    }

    @Test
    public void testValidator() {
        JSONObject jsonSchema = new JSONObject(new JSONTokener(JSONSchemaTest.class.getResourceAsStream("/schema.json")));
        JSONObject jsonSubject = new JSONObject(new JSONTokener(JSONSchemaTest.class.getResourceAsStream("/product_invalid.json")));

        Schema schema = SchemaLoader.load(jsonSchema);
        try {
            schema.validate(jsonSubject);
            System.out.println("success");
        } catch (ValidationException e) {
            System.out.println("failure");
            e.printStackTrace();
        }
    }
}
