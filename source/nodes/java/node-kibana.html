# 创建连接器

选择连接器类型, 创建连接器 默认只有两个连接器, 可用.

# 创建规则
  1. 设置监控指标查询方式
  2. 设置预警阈值

注意:

  1. 配置获知指标的方式(对应: max(urine_ph_pregnancy_days_current_parity) )
  2. 设置告警阀值 ( 对应 "高于 100" )
  3. 指标所在文档(Index) 需要有一个时间列, 来圈定时间范围内的指标(对应 "过去 5分钟" 的配置条件)
  1. 选择预设的连接器, 并配置动作
  1. 配置检查频率 (图例中, 每秒扫描一次)
  2. 选择预设的连接器( 图例中, 选择的是 "warn_info_write" 向es中写入数据 )
  3. 配置操作频率 和 操作条件
  4. 配置具体的操作的内容

# Issue:

  1. 所需监控的指标, 有哪些类型? 来源是什么?
  2. 指标监控的实时性问题?
  3. 被监控机器, 是否有控制权? (比如: SSH)
  4. 组件付费问题? (Webhook 组件需要许可证)
  5. 开发成本的问题
    • 页面汉化和修改的边界在哪?
    • 页面所有都要用吗?
  6. 不支持window原生开发，需要Docker装环境
