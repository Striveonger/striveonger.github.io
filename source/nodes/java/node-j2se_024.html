Java HashMap 底层实现原理

Mr.Lee 2023-05-30 21:26:00 Java

本节用于记录Java HashMap底层数据结构、方法实现原理等，基于JDK17

Java HashMap底层采用哈希表结构（数组+链表、JDK1.8后为数组+链表或红黑树）实现，结合了数组和链表的优点：

  1. 数组优点：通过数组下标可以快速实现对数组元素的访问，效率极高；
  2. 链表优点：插入或删除数据不需要移动元素，只需修改节点引用，效率极高。

上面这些字段在下面源码解析的时候尤为重要，其中需要着重讨论的是加载因子是什么，为什么默认值为0.75f。

加载因子也叫扩容因子，用于决定HashMap数组何时进行扩容。比如数组容量为16，加载因子为0.75，那么扩容阈值为16*0.75=12，即HashMap数据量大于等于12时，数组就会进行扩容。我们都知道，数组容量的大小在创建的时候就确定了，所谓的扩容指的是重新创建一个指定容量的数组，然后将旧值复制到新的数组里。扩容这个过程非常耗时，会影响程序性能。所以加载因子是基于容量和性能之间平衡的结果：

  • 当加载因子过大时，扩容阈值也变大，也就是说扩容的门槛提高了，这样容量的占用就会降低。但这时哈希碰撞的机率就会增加，效率下降
  • 当加载因子过小时，扩容阈值变小，扩容门槛降低，容量占用变大。这时候哈希碰撞的机率下降，效率提高。但是数组扩容频次增加, 也会导致整体性能下降

可以看到容量占用和性能是此消彼长的关系，它们的平衡点由加载因子决定，0.75是一个即兼顾容量又兼顾性能的经验值。

此外用于存储数据的 table字段使用 transient 修饰，通过 transient 修饰的字段在序列化的时候将被排除在外，那么HashMap在序列化后进行反序列化时，是如何恢复数据的呢？HashMap通过自定义的readObject/writeObject方法自定义序列化和反序列化操作。这样做主要是出于以下两点考虑：

  1. table一般不会存满，即容量大于实际键值对个数，序列化table未使用的部分不仅浪费时间也浪费空间；
  2. key对应的类型如果没有重写hashCode方法，那么它将调用Object的hashCode方法，该方法为native方法，在不同JVM下实现可能不同；换句话说，同一个键值对在不同的JVM环境下，在table中存储的位置可能不同，那么在反序列化table操作时可能会出错。

HashMap内部使用数组存储数据，数组中的每个元素类型为Node<K,V>：

/**
 * Basic hash bin node, used for most entries.  
 * (See below for TreeNode subclass, and in LinkedHashMap for its Entry subclass.)
 */
static class Node<K,V> implements Map.Entry<K,V> {
    final int hash;
    final K key;
    V value;
    Node<K,V> next;

    Node(int hash, K key, V value, Node<K,V> next) {
        this.hash = hash;
        this.key = key;
        this.value = value;
        this.next = next; // 链表的下一个节点
    }

    public final K getKey()        { return key; }
    public final V getValue()      { return value; }
    public final String toString() { return key + "=" + value; }

    public final int hashCode() {
        return Objects.hashCode(key) ^ Objects.hashCode(value);
    }

    public final V setValue(V newValue) {
        V oldValue = value;
        value = newValue;
        return oldValue;
    }

    public final boolean equals(Object o) {
        if (o == this)
            return true;
        return o instanceof Map.Entry<?, ?> e
            && Objects.equals(key, e.getKey())
            && Objects.equals(value, e.getValue());
    }
}

HashMap通过hash 方法计算key的哈希码，然后通过(n-1)&hash公式（n为数组长度）得到key在数组中存放的下标((n-1)&hash)。当两个key在数组中存放的下标一致时，数据将以链表的方式存储（哈希冲突，哈希碰撞）。我们知道，在链表中查找数据必须从第一个元素开始一层一层往下找，直到找到为止，时间复杂度为O(N)，所以当链表长度越来越长时，HashMap的效率越来越低。

为了解决这个问题，JDK1.8开始采用数组+链表+红黑树的结构来实现HashMap

当链表中的元素超过8个（TREEIFY_THRESHOLD）并且数组长度大于64（MIN_TREEIFY_CAPACITY）时，会将链表转换为红黑树，转换后数据查询时间复杂度为O(logN)

红黑树的节点使用TreeNode<K,V>表示:

/**
 * Entry for Tree bins. Extends LinkedHashMap.Entry (which in turn
 * extends Node) so can be used as extension of either regular or
 * linked node.
 */
static final class TreeNode<K,V> extends LinkedHashMap.Entry<K,V> {
    TreeNode<K,V> parent;  // red-black tree links
    TreeNode<K,V> left;
    TreeNode<K,V> right;
    TreeNode<K,V> prev;    // needed to unlink next upon deletion
    boolean red;
    
    TreeNode(int hash, K key, V val, Node<K,V> next) {
        super(hash, key, val, next);
    }

}

# HashMap.hash

HashMap通过hash方法计算key的哈希码:

static final int hash(Object key) {
    int h;
    // 该公式通过hashCode的高16位异或低16位得到哈希值, 高16位保持不变
    // 主要从性能、哈希碰撞角度考虑，减少系统开销，不会造成因为高位没有参与下标计算从而引起的碰撞。
    return (key == null) ? 0 : (h = key.hashCode()) ^ (h >>> 16);
}

# HashMap.putVal

得到key对应的哈希值后，再调用 putVal(hash(key), key, value, false, true)方法插入元素:

/**
 * Implements Map.put and related methods.
 *
 * @param hash hash for key
 * @param key the key
 * @param value the value to put
 * @param onlyIfAbsent 如果为true, 则不改变现有值
 * @param evict 在HashMap中不起作用, 在LinkedHashMap中有用到, 到时候咱们再说
 * @return previous value, or null if none
 */
final V putVal(int hash, K key, V value, boolean onlyIfAbsent, boolean evict) {
    Node<K,V>[] tab; Node<K,V> p; int n, i;
    // 如果数组(哈希表)为null或者长度为0，则进行数组初始化操作
    if ((tab = table) == null || (n = tab.length) == 0) {
        // 初始化哈希表, 并同步数组长度
        n = (tab = resize()).length;
    }
    // 根据key的哈希值计算出数据插入数组的下标位置，公式为(n-1)&hash
    if ((p = tab[i = (n - 1) & hash]) == null) {
        // 创建一个新的节点, 插入到哈希表中
        tab[i] = newNode(hash, key, value, null);
    } else {
        // 如果目标位置key已经存在, p节点(链表)
        Node<K,V> e; K k;
        if (p.hash == hash && ((k = p.key) == key || (key != null && key.equals(k)))) {
            // 当key等于链表的头节点时, e直接被覆盖
            e = p;
        } else if (p instanceof TreeNode tn) {
            // tab[i] 是红黑树时, 执行树的插入逻辑
            e = tn.putTreeVal(this, tab, hash, key, value);
        } else { // 这次put操作发生了 "哈希碰撞"
            // 在链表中查找是否key是否存在
            for (int binCount = 0; ; ++binCount) {
                // 判断是否已到链表结尾
                if ((e = p.next) == null) {
                    // 到链表尾部, 都没有找到, 说明 key不在哈希表中
                    // 新建节点, 并插入列表中采用尾插法
                    p.next = newNode(hash, key, value, null);
                    if (binCount >= TREEIFY_THRESHOLD - 1) { // -1 for 1st
                        // 链表长度 等于或超过 树化的长度阈值, 链表进行树化
                        treeifyBin(tab, hash);
                    }
                    break;
                }
                if (e.hash == hash && ((k = e.key) == key || (key != null && key.equals(k)))) {
                    // 找到key在链表中的节点
                    break;
                }
                p = e; // next node 链表指针向下继续找呗
            }
        }
        if (e != null) { // existing mapping for key
            V oldValue = e.value;
            if (!onlyIfAbsent || oldValue == null) {
                // 更新key对应节点的值
                e.value = value;
            }
            // HashMap 空方法
            afterNodeAccess(e);
            return oldValue;
        }
    }
    ++modCount;
    if (++size > threshold) {
        // 如果size超过了扩容阈值, 就需要扩容了
        resize();
    }
    // HashMap 空方法
    afterNodeInsertion(evict);
    return null;
}

# HashMap.treeifyBin

链表进行树化 treeifyBin:

/**
 * 遍历链表中的每个元素，并将其插入到红黑树中。 
 * Replaces all linked nodes in bin at index for given hash unless table is too small, in which case resizes instead.
 */
final void treeifyBin(Node<K,V>[] tab, int hash) {
    int n, index; Node<K,V> e;
    if (tab == null || (n = tab.length) < MIN_TREEIFY_CAPACITY) {
        // 当哈希表长度 < 64, 但是某个链表长度>=8了, 哈希表进行扩容并重新计算里面
        resize();
    } else if ((e = tab[index = (n - 1) & hash]) != null) { // 将链表转换为红黑树 ; 顺便给index赋值, 下面会用到的

        TreeNode<K,V> hd = null; // head
        TreeNode<K,V> tl = null; // tail
        do {
            // 替换成树节点
            TreeNode<K,V> p = replacementTreeNode(e, null);
            if (tl == null) {
                hd = p; // 初始化头节点
            } else {
                // 更新双向链表
                p.prev = tl; 
                tl.next = p;
            }
            tl = p;
        } while ((e = e.next) != null);
        //
        if ((tab[index] = hd) != null) {
            // 链表树化
            hd.treeify(tab);
        }
    }
}

# HashMap.resize

初始化或扩容哈希表时, 调用resize方法

/**
 * 扩容: 
 * 创建一个新的数组, 长度为原数组的两倍
 * 遍历原数组中的每个元素, 将它们重新计算哈希值, 并根据新数组的长度计算出在新数组中的索引
 * 
 * 根据旧哈希表的长度和扩容阈值，计算新哈希表的长度和扩容阈值。如果旧哈希表的长度为 0，则使用默认的初始容量和扩容阈值。
 * 根据新哈希表的长度，分配新的哈希表
 * 遍历旧哈希表中的每个元素，将它们重新分配到新哈希表中。如果元素的哈希值在某一位上为 0，则放在新哈希表对应的位置上；否则，放在新哈希表对应位置的后一个位置上。如果元素的哈希值相同，则放在同一个链表中。
 * 如果元素已经是红黑树节点，则调用 split 方法将其分裂成两个节点，并将这两个节点放在新哈希表中的正确位置上。
 * 
 * Initializes or doubles table size. If null, allocates in accord with initial capacity target held in field threshold.
 * Otherwise, because we are using power-of-two expansion, the elements from each bin must either stay at same index, or move with a power of two offset in the new table.
 *
 * @return the table
 */
final Node<K,V>[] resize() {
    // 一上来, 先备份下已有的哈希表
    Node<K,V>[] oldTab = table;
    // 原哈希表的长度
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    // 原哈希表的扩容阈值(默认: 16 * 0.75 = 12)
    int oldThr = threshold;
    // 预定义新哈的希表长度和扩容阈值
    int newCap, newThr = 0;
    if (oldCap > 0) { // 原哈希表的长度 > 0
        if (oldCap >= MAXIMUM_CAPACITY) {
            // 超过最大值就不再扩容了
            threshold = Integer.MAX_VALUE;
            return oldTab;
        } else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY && oldCap >= DEFAULT_INITIAL_CAPACITY) { // 注意这里哦 (newCap = oldCap << 1) 这tm也是个暴击伤害呢
            // 扩容阈值 直接暴击 * 2
            newThr = oldThr << 1; // double threshold
        }
    } else if (oldThr > 0) { // initial capacity was placed in threshold
        // 当前数组没有数据，使用初始化的值
        newCap = oldThr; // 新哈希表长度 = 原扩容阈值
    } else { // zero initial threshold signifies using defaults
        // 初始化 新哈的希表长度和扩容阈值
        newCap = DEFAULT_INITIAL_CAPACITY;  // 16
        newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY); // 12
    }
    if (newThr == 0) { // 如果新的容量等于 0
        // 重新计算 扩容阈值
        float ft = (float)newCap * loadFactor;
        newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ? (int)ft : Integer.MAX_VALUE);
    }
    // 扩容阈值 确定下来了~
    threshold = newThr;
    @SuppressWarnings({"rawtypes","unchecked"})
    // 原数组长度 * 2
    Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap]; // 内存开辟新的空间
    table = newTab; // 如果 初始化来的, 这到已经算是功德圆满了...
    if (oldTab != null) { // 重新根据key.hash计算在哈希表中的索引, 毕竟 n 暴击了嘛...
        for (int j = 0; j < oldCap; ++j) {
            Node<K,V> e;
            if ((e = oldTab[j]) != null) {
                // 先删除数组作用在 node对象身上的指针
                oldTab[j] = null; // 在并发情况下, 线程在这被挂起了, 这时来了个put,好死不死的命中了这个索引, 成功入驻链表的头节点...高兴了没一会儿呢, 走到...
                if (e.next == null) {
                    // 如果链表中只有一个节点，则进行直接赋值
                    newTab[e.hash & (newCap - 1)] = e;
                } else if (e instanceof TreeNode tn) {
                    // 当前为树节点, 执行树拆分逻辑
                    tn.split(this, newTab, j, oldCap);
                } else { // preserve order 链表中有多个节点时
                    // 这里把链表一分为二...
                    Node<K,V> loHead = null, loTail = null; // e.hash & oldCap == 0
                    Node<K,V> hiHead = null, hiTail = null; // e.hash & oldCap <> 0
                    Node<K,V> next;
                    do {
                        // 根据规则分就完了...
                        next = e.next;
                        if ((e.hash & oldCap) == 0) { 
                            if (loTail == null) {
                                loHead = e;
                            } else {
                                loTail.next = e;
                            }
                            loTail = e;
                        } else {
                            if (hiTail == null) {
                                hiHead = e;
                            } else {
                                hiTail.next = e;
                            }
                            hiTail = e;
                        }
                    } while ((e = next) != null);
                    // 把拆分后的链表, 重新挂载到
                    if (loTail != null) {
                        loTail.next = null;
                        newTab[j] = loHead;
                    }
                    // 将原索引 + oldCap 放到哈希表中
                    if (hiTail != null) {
                        hiTail.next = null;
                        newTab[j + oldCap] = hiHead;
                    }
                }
            }
        }
    }
    return newTab;
}

# TreeNode.treeify

链表树化的逻辑 TreeNode.treeify:

/**

 * Forms tree of the nodes linked from this node.
 */
final void treeify(Node<K,V>[] tab) {
    TreeNode<K,V> root = null;
    for (TreeNode<K,V> x = this, next; x != null; x = next) { // 开始循环链表
        next = (TreeNode<K,V>) x.next;
        x.left = x.right = null;
        if (root == null) {
            x.parent = null;
            x.red = false;
            root = x;
        } else {
            K k = x.key;
            int h = x.hash;
            Class<?> kc = null;
            for (TreeNode<K,V> p = root;;) {
                int dir, ph;
                K pk = p.key;
                // 获得正确的dir
                if ((ph = p.hash) > h) {
                    dir = -1;
                } else if (ph < h) {
                    dir = 1;
                } else if ((kc == null && (kc = comparableClassFor(k)) == null) || (dir = compareComparables(kc, k, pk)) == 0) {
                    dir = tieBreakOrder(k, pk);
                }

                TreeNode<K,V> xp = p;
                if ((p = (dir <= 0) ? p.left : p.right) == null) {
                    x.parent = xp;
                    if (dir <= 0) {
                        xp.left = x;
                    } else {
                        xp.right = x;
                    }
                    root = balanceInsertion(root, x);
                    break;
                }
            }
        }
    }
    moveRootToFront(tab, root);
}

/**
 * 更新红黑树
 * Tree version of putVal.
 */
final TreeNode<K,V> putTreeVal(HashMap<K,V> map, Node<K,V>[] tab, int h, K k, V v) {
    Class<?> kc = null;
    boolean searched = false;
    TreeNode<K,V> root = (parent != null) ? root() : this;
    for (TreeNode<K,V> p = root;;) {
        int dir, ph; K pk;
        if ((ph = p.hash) > h)
            dir = -1;
        else if (ph < h)
            dir = 1;
        else if ((pk = p.key) == k || (k != null && k.equals(pk)))
            return p;
        else if ((kc == null &&
                  (kc = comparableClassFor(k)) == null) ||
                 (dir = compareComparables(kc, k, pk)) == 0) {
            if (!searched) {
                TreeNode<K,V> q, ch;
                searched = true;
                if (((ch = p.left) != null &&
                     (q = ch.find(h, k, kc)) != null) ||
                    ((ch = p.right) != null &&
                     (q = ch.find(h, k, kc)) != null))
                    return q;
            }
            dir = tieBreakOrder(k, pk);
        }

        TreeNode<K,V> xp = p;
        if ((p = (dir <= 0) ? p.left : p.right) == null) {
            Node<K,V> xpn = xp.next;
            TreeNode<K,V> x = map.newTreeNode(h, k, v, xpn);
            if (dir <= 0)
                xp.left = x;
            else
                xp.right = x;
            xp.next = x;
            x.parent = x.prev = xp;
            if (xpn != null)
                ((TreeNode<K,V>)xpn).prev = x;
            moveRootToFront(tab, balanceInsertion(root, x));
            return null;
        }
    }
}

/**
 * Splits nodes in a tree bin into lower and upper tree bins, or untreeifies if now too small. Called only from resize;
 * see above discussion about split bits and indices.
 *
 * @param map the map
 * @param tab the table for recording bin heads
 * @param index the index of the table being split
 * @param bit the bit of hash to split on
 */
final void split(HashMap<K,V> map, Node<K,V>[] tab, int index, int bit) {
    TreeNode<K,V> b = this;
    // Relink into lo and hi lists, preserving order
    TreeNode<K,V> loHead = null, loTail = null;
    TreeNode<K,V> hiHead = null, hiTail = null;
    int lc = 0, hc = 0;
    for (TreeNode<K,V> e = b, next; e != null; e = next) {
        next = (TreeNode<K,V>)e.next;
        e.next = null;
        if ((e.hash & bit) == 0) {
            if ((e.prev = loTail) == null)
                loHead = e;
            else
                loTail.next = e;
            loTail = e;
            ++lc;
        }
        else {
            if ((e.prev = hiTail) == null)
                hiHead = e;
            else
                hiTail.next = e;
            hiTail = e;
            ++hc;
        }
    }

    if (loHead != null) {
        if (lc <= UNTREEIFY_THRESHOLD)
            tab[index] = loHead.untreeify(map);
        else {
            tab[index] = loHead;
            if (hiHead != null) // (else is already treeified)
                loHead.treeify(tab);
        }
    }
    if (hiHead != null) {
        if (hc <= UNTREEIFY_THRESHOLD)
            tab[index + bit] = hiHead.untreeify(map);
        else {
            tab[index + bit] = hiHead;
            if (loHead != null)
                hiHead.treeify(tab);
        }
    }
}

/**
 * Implements Map.get and related methods.
 *
 * @param key the key
 * @return the node, or null if none
 */
final Node<K,V> getNode(Object key) {
    Node<K,V>[] tab; Node<K,V> first, e; 
    int n, hash; K k;
    // 初始化参数 & 判空
    if ((tab = table) != null && (n = tab.length) > 0 && (first = tab[(n - 1) & (hash = hash(key))]) != null) {
        // 检查第一个key.hash命中的节点
        if (first.hash == hash && ((k = first.key) == key || (key != null && key.equals(k)))) {
            // 幸运命中返回
            return first;
        }
        if ((e = first.next) != null) { // 判断是否有后续节点
            if (first instanceof TreeNode tn) {
                // 如果命中节点为树节点, 则执行红黑树的查询逻辑
                return tn.getTreeNode(hash, key);
            }
            do {
                // 链表依次下去吧~
                if (e.hash == hash && ((k = e.key) == key || (key != null && key.equals(k)))) {
                    return e;
                }
            } while ((e = e.next) != null);
        }
    }
    return null;
}

# TreeNode.find

/**
 * O(N*LogN)
 * Finds the node starting at root p with the given hash and key.
 * The kc argument caches comparableClassFor(key) upon first use comparing keys.
 */
final TreeNode<K,V> find(int h, Object k, Class<?> kc) {
    TreeNode<K,V> p = this;
    do {
        int ph, dir; K pk;
        TreeNode<K,V> pl = p.left, pr = p.right, q;
        if ((ph = p.hash) > h) {
            p = pl;
        } else if (ph < h) {
            p = pr;
        } else if ((pk = p.key) == k || (k != null && k.equals(pk))) {
            return p;
        } else if (pl == null) {
            p = pr;
        } else if (pr == null) {
            p = pl;
        } else if ((kc != null || (kc = comparableClassFor(k)) != null) && (dir = compareComparables(kc, k, pk)) != 0) {
            p = (dir < 0) ? pl : pr;
        } else if ((q = pr.find(h, k, kc)) != null) {
            return q;
        } else {
            p = pl;
        }
    } while (p != null);
    return null;
}

final class EntrySet extends AbstractSet<Map.Entry<K,V>> {
    public final int size()                 { return size; }
    public final void clear()               { HashMap.this.clear(); }
    public final Iterator<Map.Entry<K,V>> iterator() {
        return new EntryIterator();
    }
    public final boolean contains(Object o) {
        if (!(o instanceof Map.Entry<?, ?> e))
            return false;
        Object key = e.getKey();
        Node<K,V> candidate = getNode(key);
        return candidate != null && candidate.equals(e);
    }
    public final boolean remove(Object o) {
        if (o instanceof Map.Entry<?, ?> e) {
            Object key = e.getKey();
            Object value = e.getValue();
            return removeNode(hash(key), key, value, true, true) != null;
        }
        return false;
    }
    public final Spliterator<Map.Entry<K,V>> spliterator() {
        return new EntrySpliterator<>(HashMap.this, 0, -1, 0, 0);
    }
    public final void forEach(Consumer<? super Map.Entry<K,V>> action) {
        Node<K,V>[] tab;
        if (action == null)
            throw new NullPointerException();
        if (size > 0 && (tab = table) != null) {
            int mc = modCount;
            for (Node<K,V> e : tab) {
                for (; e != null; e = e.next)
                    action.accept(e);
            }
            if (modCount != mc)
                throw new ConcurrentModificationException();
        }
    }
}

# HashIterator

abstract class HashIterator {
    Node<K,V> next;        // next entry to return
    Node<K,V> current;     // current entry
    int expectedModCount;  // for fast-fail
    int index;             // current slot

    HashIterator() { // 初始化迭代器
        expectedModCount = modCount;
        Node<K,V>[] t = table;
        current = next = null;
        index = 0;
        if (t != null && size > 0) { // advance to first entry
            // 在哈希表中找到第一个不为空的节点
            do { } while (index < t.length && (next = t[index++]) == null);
        }
    }

    public final boolean hasNext() {
        // 直接判空就完了
        return next != null;
    }

    final Node<K,V> nextNode() {
        Node<K,V>[] t;
        Node<K,V> e = next;
        // 迭代过程中, 向哈希表中增加或删除过节点
        if (modCount != expectedModCount) { throw new ConcurrentModificationException(); }
        if (e == null) { throw new NoSuchElementException(); }
        
        // 赋值 并判空, 链表没走完, 就不会进下一个链表
        if ((next = (current = e).next) == null && (t = table) != null) { // 秀儿, 是你嘛~
            // 向下找, 下一个不为空的链表 表头
            do {} while (index < t.length && (next = t[index++]) == null);
        }
        return e;
    }

    public final void remove() {
        Node<K,V> p = current;
        if (p == null) { throw new IllegalStateException(); }
        if (modCount != expectedModCount) { throw new ConcurrentModificationException(); }
        current = null;
        removeNode(p.hash, p.key, null, false, false);
        expectedModCount = modCount;
    }
}
